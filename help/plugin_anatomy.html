Anatomy of a plugin

There are 3 types of plugins: payment, shipping and order total. 2 of them act exactly the same; payment and shipping while order total is slightly diferent. The reason being is that order total impacts the total amount so it needs to be processed when adding an item to the cart. This way the cart total takes into account any order total plugin.

  1. Payment plugins- They provide your users with a means to pay for items.
  2. Shipping plugins- This provides delivery methods.
  3. Order Total plugins- Impacts the users cart or/and cart total.

plugin_version.php This file provides all the relevant info regarding the plugin:

  • $plugin['name'] Name of plugin
  • $plugin['version'] Version number og plugin
  • $plugin['description'] A brief description of what this plugin does.
  • $plugin['dirname'] Directory of plugin
  • $plugin['type'] Can be either payment, shipping or order total
  • $plugin['sqlfile'] sql file which will create tables and insert data.
  • $plugin['tables'][0] Table name. [0] should be incremented by 1 for each additional table

mysql.sql Creates tables for your plugin.In order for kshop to correctly install and uninstall, your plugin tables should use the following standard: kshop_plug_<dir_name>_table_name.

settings.php Allows you to create options and preferences that are stored in it´s own table inside the DB. This will be displayed in a popup window when the icon "settings" is clicked in the plugin tab.

templates/step.tpl.php Displays a table with a radio button. There are a couple of do's and don'ts regarding this file:

Don't:

  • Don't change the name of the radio button otherwise the plugin won't work.
  • Don't make the table a LARGE fixed width. There are sites that use 750pix leaving about 450 pix for the center blocks.

Do:

  • Make the table informative. Let people know what exactly does this plugin offers.
  • Keep the graphics light. Use images to make it more visually appealing but remeber to optimize them for web use to keep their size low.


process.php If your module needs to add, subtract,etc to the total price, this is where it happens. This applies to the cart and checkout process. When a user clicks finalize order, kshop will perform some tasks for each plugin used in the order (shipping, payment and order total). If everything is ok it will continue with process, if an error ocurred, display error messsage, reason and exit process order.

language/english This directory contains your language file. If your Xoops install uses a diferent language, you should create a language file with the same name as your xoops language for each plugin. If kshop can't find your language file it will default to the english version, so it's important you leave it in as a safe guard.
