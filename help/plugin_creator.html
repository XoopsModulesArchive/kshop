Plugin Creator Guide.

To create a new plugin it's better to base it off an existing plugin such as payment on delivery to use as a reference. With that said let's get starting:

  1. Define what type of plugin you want to create. It can be either a payment, shipping or order total.
  2. Understand what exactly the plugin should do. When building a plugin it's important you know exactly what it's supposed to do. It's preferable to start with a smaller scope and then increase from that, then try to build a plugin that's to vast and get lost in it's settings.
  3. Define your plugin name and it's base directory. It's important to keep the directory name unique so that it won't conflict with other plugins.
  4. Choose names for your table(s). A good practice would be to name your tables as: kshop_plug_<dir_name>_table_name This will make it easier to find and edit in phpmyadmin
  5. Update plugin_version.php with your new info.
  6. Create the necessary fields in mysql.sql If your module just uses a single set of settings, then 1 table will do. Then just update those fields whenever an admin clicks on setings.
  7. Create settings in settings.php
  8. Design your template files

To better understand plugins I adivise you open banktransfer and read it's commented code.
